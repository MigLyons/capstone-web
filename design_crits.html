Design Crits

Once we had a stable high-fidelity prototype, we wanted to observe how other students in our class (and potential end users of our app) would perform a set of common tasks using our prototoype. In particular, we wanted to learn where they had questions, problems, and concerns with our design. Below is the set of scenarios/tasks that we provided to our classmates during this high-fidelity design crit session. This is followed by some of the most important feedback we gathered and a discussion of how it will impact our future design iterations.

Scenario 1

A team lead is putting together a group of consultants for a new project. They want to search Cognizant profiles to find consultants with skill sets to match the project scope.

Tasks:

  1. Login
  2. Navigate to the search screen
  3. Search for “Salesforce Developer”
  4. Click “Need Help” for AI-generated help available to guide search terms
  5. Filter the subsections you would like be considered in your search
  6. View the matching consultant profiles

Scenario 2

A consultant would like to create a new profile manually within the application.

Tasks:

  1. Login
  2. Navigate to ‘Manage Profiles’ to manually create a profile
  3. Enter user information into the subsections of the profile
  4. Save profile

Scenario 3

A consultant would like to update one of the existing profiles.

Tasks:

  1. Login
  2. Navigate to ‘Manage Profiles’
  3. Select ‘Edit’ on a profile from list of current profiles to update
  4. Update desired subsections of the profile
  5. Save profile

Summary of feedback from in-class Design Crits

Below is a list of the most common feedback we received during our design crit session using our high-fidelity prototype.

  • Logo: Change the image used for the Cognizant logo, center of logo should be transparent
  • Change Button Text: Change “Need Help” button to used terminology that would more so imply its function to be an AI-assistant tool
  • Search Button Arrangement: Change the “Search,” “Filter,” and “Need Help” to not be stacked on top of each other
  • Button Icons: Change look of archive button, potentially add a trash icon to signal to the user the action would essentially no longer display that profile within the UI
  • Export Button Arrangement: Have the export button be moved to the top, rather then on the bottom
  • Profile Section Dropdowns: Have the dropdowns be defaulted to expanded, and allow the user to dismiss or hide specific information. Rather then having the dropdowns be defaulted to hidden
To illustrate how this feedback will impact future iterations of our design, we included an annotated screenshot illustrating the problems, alongside a description of how we plan to resolve these particular issues going forward.
  • Button Arrangement: Place the "Need Help" button underneath the search bar and place the "Filter" button beside the "Search" button. This will make finding the buttons more intuitive rather then having them stacked upon each other.
  • Contrast: Making the search bar and background have more contrast to easily see where the borders of the search bar are. Also, created more contrast with the search input text and background of search bar so user can easily read what they have typed.
  • Navigation: Add a back button that links to the main page. This functionality can be added either through an additional button or by clicking the Cognizant logo.
