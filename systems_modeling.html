Systems Modeling

In the process of developing our system, we needed a way to visualize how different parts of our architecture are related and interact. We chose to create two core models: an Entity-Relationship (ER) Diagram to show how profile data is structured and related, and a System Workflow Diagram to illustrate the end-to-end movement of profile data through Azure services and internal components.

Entity-Relationship Diagram

This diagram outlines the structure of our database, particularly the relationship between profiles, their contents, and associated section categories. It's used to ensure proper relational mapping and normalization of consultant data.

It shows how a profile contains multiple section contents, and each section content is identified by a section category. This structure allows dynamic parsing and querying of profile data.

System Workflow Diagram

This diagram details the data flow between SharePoint, Azure services, and our web app. It helps visualize automation pipelines and how updates to profiles trigger our backend API and storage operations.

From uploading PowerPoint profiles in SharePoint, to transforming them into JSON, storing in Azure Blob, and finally pushing to an Azure SQL database, this diagram illustrates how each part connects to support search and export features.
