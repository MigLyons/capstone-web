High-fidelity Prototype

After presenting our low-fidelity prototypes to class and to our clients, we received numerous suggestions for improvement as summarized below:

  • Profile information displayed in dropdown boxes that are collapsible
  • Export/download feature should have a preview page
  • Checkboxes for filtering selection rather than radio buttons
  • Center search box vertically in the search screen

After considering and incorporating most of the feedback we received into our design, we then set out to create a high-fidelity prototype. High-fidelity prototypes are more developed than the low-fidelity UI. Our high-fidelity prototype is a model of our web app that is navigable, but does not have a back-end or live data. We chose to use Figma to develop our high-fidelity prototype because it provides extremely developed and high definition app components, and it has free functionality.

Below are some screen shots taken from our high-fidelity prototype.They depict the login screen, main screen, search screen, search results screen, profile view screen, and export preview screen. Through this flow of screens, we show the flow of logging in, searching for a consultant matching some criteria, finding such a consultant, and exporting their profile for further reference.

These screens show the login screen followed by the main screen, which is a static flow. It continues as though the user had selected "Search Profiles", and then searched for "Salesforce Developer". The search results screen shows the results of the search, and the profile view screen shows the profile data of a consultant in collapsible and easy-to-digest segments. The export preview screen shows the whole formatted profile of a consultant that can be exported to a PDF.

These screens show the main screen, and then continues as though the user had selected "Manage Profiles". The "Manage Profiles" page provides options to manually create a profile through the creation form, or to upload a profile.
